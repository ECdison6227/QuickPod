# Contributing to QuickPod

Thanks for your interest in improving QuickPod.

## How to contribute

1. Open an [Issue](https://github.com/ECdison6227/QuickPod/issues) to describe the bug or feature first.
2. Fork the repo and create a branch.
3. Make your changes.
4. Run `./build.sh` and make sure the app compiles.
5. Open a Pull Request with a clear description and screenshots if the UI changed.

## Development setup

```bash
git clone https://github.com/ECdison6227/QuickPod.git
cd QuickPod
./build.sh
```

## Code style

- Swift code follows the existing formatting in `Sources/`.
- Keep menu-bar-only interactions lightweight.
- New permissions must be clearly documented in README and permissions sections.
