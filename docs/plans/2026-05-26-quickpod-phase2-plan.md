# QuickPod 项目二阶段更改计划

## 一、计划概述

### 1.1 项目背景
QuickPod 是一款 macOS 平台的效率工具应用，一阶段已完成核心功能开发，包括防睡眠管理、休息提醒、屏幕清洁、快速切换器等模块。二阶段将聚焦于**交互体验优化**、**代码质量提升**和**用户引导完善**，为用户提供更加流畅、直观的使用体验。

### 1.2 计划目标
| 目标维度 | 具体目标 |
|---------|---------|
| 功能完善 | 优化状态栏交互、增强快捷操作能力 |
| 代码质量 | 代码覆盖率≥80%，消除所有警告 |
| 用户体验 | 开发引导动画，降低学习成本 |
| 项目管理 | 建立规范的 Git 工作流 |
| 交付时间 | 2026年6月30日前完成所有任务 |

## 四、Git 仓库建立与管理

### 4.1 分支管理策略

```
main (主分支，稳定版本)
├── develop (开发分支，集成所有功能)
│   ├── feature/* (功能开发分支)
│   ├── bugfix/* (Bug修复分支)
│   └── hotfix/* (紧急修复分支)
└── release/* (发布准备分支)
```

**分支命名规范：**
- `feature/xxx` - 新功能开发
- `bugfix/xxx` - Bug 修复
- `hotfix/xxx` - 紧急线上修复
- `release/x.y.z` - 版本发布准备

### 4.2 提交规范

采用 Conventional Commits 格式：

```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

**Type 类型：**
- `feat` - 新功能
- `fix` - Bug 修复
- `docs` - 文档更新
- `style` - 代码格式（不影响代码运行）
- `refactor` - 重构（既不新增功能也不修复 Bug）
- `test` - 测试相关
- `chore` - 构建/工具更新

### 4.3 代码审查流程

1. **创建 PR**：从 feature/bugfix 分支提交到 develop
2. **自动检查**：CI 自动运行测试和静态分析
3. **代码审查**：至少1位 reviewer 审核通过
4. **合并**：使用 Squash Merge 合并到 develop
5. **删除分支**：合并后自动删除源分支

### 4.4 版本控制流程

```
开发 → 测试 → 发布候选 → 发布 → 归档
   ↑         ↑            ↑
  feature  bugfix     release
```

## 五、代码与资源优化

### 5.1 优化范围

| 类别 | 优化内容 | 优先级 |
|-----|---------|-------|
| 代码清理 | 移除未使用的变量、函数、类 | 高 |
| 资源优化 | 删除冗余图片、过期配置文件 | 高 |
| 结构优化 | 重构模块划分，优化目录结构 | 中 |
| 依赖清理 | 移除未使用的第三方依赖 | 中 |

### 5.2 优化标准

| 检查项 | 标准 | 检查工具 |
|-------|-----|---------|
| 未使用导入 | 无 import 未使用 | SwiftLint |
| 未使用变量 | 无警告提示 | Xcode 编译器 |
| 死代码 | 代码覆盖率为0的路径 | Xcode Coverage |
| 资源冗余 | 无未引用的资源文件 | 手动检查 |

## 六、引导动画开发

### 6.1 技术方案

**技术选型**：Remotion + SwiftUI 动画框架

**集成方式**：
- 使用 Remotion 生成引导动画视频
- 在应用启动时通过 AVPlayer 播放
- 或使用 SwiftUI 原生动画实现交互式引导

### 6.2 引导场景设计

| 场景 | 内容 | 时长 |
|-----|-----|-----|
| 场景1 | 介绍应用定位和核心价值 | 3秒 |
| 场景2 | 演示状态栏图标交互（左键/右键） | 4秒 |
| 场景3 | 演示防睡眠功能开关 | 3秒 |
| 场景4 | 演示屏幕清洁功能 | 3秒 |
| 场景5 | 演示快速切换器 | 3秒 |

### 6.3 交互设计

| 交互类型 | 设计要点 |
|---------|---------|
| 点击引导 | 高亮可点击区域，显示手势提示 |
| 状态变化 | 使用过渡动画展示功能效果 |
| 进度指示 | 显示引导进度，支持跳过/暂停 |
| 自动播放 | 首次启动自动播放，可随时跳过 |

### 6.4 验收标准

| 指标 | 标准 |
|-----|-----|
| 动画完整性 | 覆盖所有核心功能引导 |
| 播放流畅度 | 无卡顿、无丢帧 |
| 用户理解度 | 新用户看完后能独立完成基本操作 |
| 可跳过性 | 支持随时跳过引导 |
